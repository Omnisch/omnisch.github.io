UE5 TUIO 插件简明文档

发表于 2025-03-18 分类于 pen

RayionTuioUe5 插件改自 UE4_TUIO_2101143，在升级至 UE5 的基础上大幅修改了接口并添加了新功能。

原插件注意事项如下：

C++ 类

AMyTuioActor

AMyTuioActor 监听到添加令牌的消息时，会调用 AddMarker(FMarkerInfo)，参数包含了被添加的令牌的信息，其他同理。

蓝图 Actor 类

TuioListener

由于工程历史原因，此 TuioListener 与 TUIO 源代码中的 TuioListener.h 没有关系。

TuioListener 位于 /Plugins/RayionTuioUe5 Content/Blueprints/，继承自 AMyTuioActor，实现了 AMyTuioActor 关于令牌与触点的方法。对于令牌，TuioListener 会向对应 Marker 的 BPI_TuioListenerDispatcher 接口发送消息；对于触点，TuioListener 会维护 MyUiCursor 列表，监听触点消息创建、更新、销毁 MyUiCursor。

MarkerObject_base

MarkerObject_base 位于 /Plugins/RayionTuioUe5 Content/Markers/，是所有 Marker 的基类，继承并实现接口 BPI_TuioListenerDispatcher。

基本功能为在世界空间中随 TUIO 令牌消息而改变 Transform，位置的 (0,0) 和 (1,1) 两点由 TuioListener 开放在 Details 面板的 PosMin 和 PosMax 两物体决定。

MyUiCursor

UE5 引擎原生只支持最多 10 个触点，显然无法满足 TUIO 使用，所以需要通过 TUIO 传递触点并在引擎中解析、模仿光标的操作。然而若自行编译引擎调高触点上限，就不再需要 MUCR 了。

MyUiCursor 简称 MUCR，位于 /Plugins/RayionTuioUe5 Content/Markers/。每帧查找场景中所有的 BPI_MyUiCursorReceiver 并发送 OnTouchMove() 等消息。只存在于一次触点会话中，触点抬起即销毁。
